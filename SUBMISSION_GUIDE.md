# Submission Guide

## Recommended Submission

Send the live demo link and GitHub repository link. Attach the zip as a backup.

```text
Live prototype:
https://YOUR-GITHUB-USERNAME.github.io/aspira-open-house-redesign/

GitHub repository:
https://github.com/YOUR-GITHUB-USERNAME/aspira-open-house-redesign
```

## Why This Format Fits The Role

The take-home assessment allows a custom-coded page, and the job posting focuses on WordPress, CMS support, semantic HTML, website QA, and marketing-led website improvements.

This submission therefore includes:

- A static prototype that reviewers can open immediately.
- A WordPress-ready template folder that shows how the page can be adapted into Sienna/Aspira's WordPress environment.
- Notes explaining UX, brand, accessibility, and conversion decisions.

## What To Upload To GitHub

Upload the contents of the `aspira-open-house` folder:

```text
index.html
styles.css
script.js
README.md
SUBMISSION_GUIDE.md
notes.md
assets/
wordpress-template/
```

Do not upload the original task folder, the brand guideline PDF, the saved HubSpot webpage files, or tracking scripts.
